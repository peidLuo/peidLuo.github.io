call和apply看起来只是参数的不同，明确知道参数时使用call，否则使用apply。
理解了使用方法后我们再来看权威的书面表达。

MDN描述

根据网上各位大牛的说法，性能上call比apply要好一些，最新的引擎则基本相同，我们简单的测试一下。

可以看出，call和apply在浏览器的确有性能上的差异，chrome下表现较为明显。因此在call和apply都能使用的场景下，call应当是我们的优选。

后来在大佬的测试文章里看到。经测试，发现Array.prototype.slice.apply的确略快于 Array.prototype.slice.call，因此上面的结论有失水准，call和apply看情况使用即可。

经典用法

最后这篇文章的总结很不错。

apply 、 call 、bind 三者都是用来改变函数的this对象的指向的；
apply 、 call 、bind 三者第一个参数都是this要指向的对象，也就是想指定的上下文；
apply 、 call 、bind 三者都可以利用后续参数传参；
bind 是返回对应函数，便于稍后调用；apply 、call 则是立即调用 。
